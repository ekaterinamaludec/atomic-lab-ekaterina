# Анализ рисков подхода с неатомарными коммитами

## Основные риски:

1. **Сложность отката (revert)** — при неатомарном коммите откатить нужно все изменения сразу

2. **Проблемы при код-ревью** — сложно понять, какое изменение к чему относится

3. **Риск «снежного кома» зависимостей** — изменения переплетаются

4. **Проблемы с git bisect** — при поиске бага бисект указывает на большой коммит
